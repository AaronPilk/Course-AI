# Launch windows + ops planning

## Launch window math

For LEO: Earth + payload + orbital plane must align. ~10-30 min daily window typical.

For interplanetary: Earth + destination geometry. Window often weeks long (24 days for Mars 2026).

For: planning operations.

## Mission phases

1. **Pre-launch.** Years of design, integration, test.
2. **Launch.** Minutes; nail-biting.
3. **Early orbit.** Days; deploy + checkout.
4. **Commissioning.** Weeks; instruments calibrated.
5. **Science / operations.** Months to decades.
6. **Decommissioning.** End of life.

For: timeline understanding.

## Mission control

Where ops happen:
- **NASA JPL.** Deep space + Mars.
- **NASA JSC (Houston).** Crewed.
- **NASA GSFC.** Earth science.
- **SpaceX HQ.** Falcon + Dragon.
- **ESA ESOC (Darmstadt).** ESA missions.

24/7 staffing during active phases.

For: operations geography.

## Ops shift structure

- **Flight Director.** Highest authority, real-time.
- **Capcom (crewed).** Talks to crew.
- **System officers.** Per subsystem.
- **Public affairs.** Communicates outside.

NASA mission control rooms iconic.

For: roles.

## Daily ops

For active mission:
- Plan next-day commands.
- Review yesterday's telemetry.
- Adjust based on instrument data.
- Long-term planning (weekly, monthly).

Repetitive but high-stakes.

For: daily rhythm.

## Mid-course correction (MCC)

Trajectory rarely perfect. Adjust with small burns:
- Voyager: many MCCs over decades.
- Mars probes: 4-6 corrections en route typically.
- Apollo: precision required for free return.

For: navigation refinement.

## Telemetry

Spacecraft sends streaming data:
- House-keeping (temperatures, power, attitudes).
- Engineering (subsystem status).
- Science (instrument data).

Rates vary by spacecraft + distance.

For: situation awareness.

## Anomaly response

When something goes wrong:
- Spacecraft enters safe mode.
- Telemetry analyzed.
- Reproduce in simulator if possible.
- Plan recovery.

Famous: Galileo HGA stuck (worked with LGA backup); Hubble mirror error (added correction optics); Voyager AC fixes from 24B km.

For: troubleshooting.

## Mission planning tools

- **Sequence files.** Pre-programmed commands for periods.
- **Schedulers.** STK, SPADE.
- **Simulators.** Verify before sending.
- **What-if tools.** Analyze contingencies.

For: ops infrastructure.

## Decision-making

Real-time decisions:
- Press to MECO (Main Engine Cut-Off)?
- Continue burn vs. abort?
- Activate science vs. troubleshoot?

Often time-pressured + irreversible.

For: pressure environment.

## Mistakes to avoid

- **No simulator runs.** Untested commands.
- **Insufficient ground rehearsal.** Real ops fails.
- **Single-person decision.** Need cross-checks.

## Summary

- Launch windows constrain departure.
- Mission phases: pre-launch through decommission.
- 24/7 ops; multiple control rooms.
- Mid-course corrections + safe modes for issues.
- Sequence files + simulators are ops infrastructure.
