# UI — UGUI and UI Toolkit

## Two UI systems

Unity has two UI systems in 2026:
- **UGUI (Unity GUI).** Older; component-based; massive marketplace + tutorial coverage.
- **UI Toolkit.** Newer; web-style (HTML/CSS-like); growing adoption; better performance.

For learning: UGUI (more resources). For new projects: UI Toolkit recommended (Unity's future direction).

## UGUI fundamentals

UGUI uses Canvas + UI components on GameObjects in scene.

Create: GameObject menu → UI → Canvas. Adds:
- Canvas (renders UI).
- EventSystem (handles input).

Then add child UI elements: Text, Image, Button, etc.

## Canvas

The container for UI:
- **Render Mode.** Screen Space - Overlay (default; UI on screen), Screen Space - Camera (UI rendered by camera), World Space (UI in 3D world).
- **UI Scale Mode.** Constant Pixel Size, Scale With Screen Size, Constant Physical Size.

For most games: Screen Space - Overlay + Scale With Screen Size.

## Common UI components

- **Text / TextMeshPro.** Display text. TMPro recommended (cleaner rendering).
- **Image.** Display a sprite.
- **Button.** Interactive; OnClick events.
- **Slider.** Range input.
- **Toggle.** Checkbox.
- **InputField.** Text input.
- **Dropdown.** Select from options.
- **Scroll View.** Scrollable content.

For: standard UI building blocks.

## RectTransform

UI version of Transform; positioned in 2D:
- **Anchor points.** How positions relate to parent.
- **Pivot.** Local origin.
- **Position / Size.** Relative to anchors.

For: responsive UI that scales with screen.

## Anchors

Critical for responsive UI:
- **Top-Left.** Position relative to top-left corner.
- **Center.** Relative to center.
- **Stretch.** Stretches with parent.

For: UI that adapts to different aspect ratios.

## Layouts

For: structured UI:
- **Vertical Layout Group.** Stacks children vertically.
- **Horizontal Layout Group.** Stacks horizontally.
- **Grid Layout Group.** Grid arrangement.
- **Content Size Fitter.** Auto-sizes based on content.

For: dynamic UI (lists, inventories).

## Button OnClick

```csharp
button.onClick.AddListener(MyMethod);
```

Or in Inspector: drag target GameObject + method.

For: button-driven interactions.

## TextMeshPro

Replaces legacy Text:
- Better rendering quality.
- Rich text support.
- Better performance.

Import via Window → TextMeshPro → Import TMP Essentials.

For: all text in modern Unity UI.

## UI Toolkit

Newer system:
- **UXML.** Layout (like HTML).
- **USS.** Styles (like CSS).
- **C# logic.** Behavior.

Create UI Documents; reference USS stylesheets.

For: complex UI; better performance; designer-friendly via UI Builder.

## UI animations

- **Animator** on UI GameObject for state-based animation.
- **DOTween** library for tweens (free + paid).
- **LeanTween** lightweight alternative.

For: smooth UI transitions; modern feel.

## World-space UI

For UI attached to 3D world objects (health bars over enemies, name tags):
- Canvas Render Mode = World Space.
- Position the Canvas as 3D object.
- Often face camera via script.

For: diegetic UI; 3D HUD elements.

## Common patterns

**Main menu:** Canvas + buttons (Play, Options, Quit).
**HUD:** Canvas + Text (score) + Image (health bar fill).
**Pause menu:** activate when paused; deactivate when resuming.
**Inventory:** Scroll View + dynamically-spawned item buttons.

For: standard game UI structure.

## Mistakes to avoid

- **Legacy Text.** Use TextMeshPro.
- **Hardcoded positions.** Won't scale with screen.
- **No anchor setup.** UI breaks on different resolutions.
- **Heavy Canvas redraws.** Performance hit.
- **All UI on one Canvas.** Redrawing entire UI for any change.

## Summary

- UGUI = component-based; UI Toolkit = web-style.
- Canvas + UI elements + EventSystem.
- RectTransform with anchors for responsive layout.
- Layout Groups for structured UI.
- TextMeshPro for all text.
- Button OnClick events.
- World-space UI for 3D-attached.
