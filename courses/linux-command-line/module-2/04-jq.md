# jq: JSON in the shell

## Why jq

APIs return JSON. Logs are JSON. Configs are JSON. Without a real JSON parser, you'd be grepping inside structured data and mangling it. `jq` is the standard tool: a JSON-aware filter.

```
$ curl -s api.github.com/users/torvalds | jq '.name'
"Linus Torvalds"
```

Install:

```
$ apt install jq      # Debian/Ubuntu
$ brew install jq     # macOS
$ dnf install jq      # Fedora
```

## Basic queries

`.` is the entire input. `.field` selects a field:

```
$ echo '{"name": "Linus", "age": 55}' | jq '.name'
"Linus"

$ echo '{"user": {"name": "Linus"}}' | jq '.user.name'
"Linus"
```

`-r` for raw output (no quotes):

```
$ echo '{"name": "Linus"}' | jq -r '.name'
Linus
```

`.[0]` for array indexing:

```
$ echo '[1,2,3]' | jq '.[0]'
1
```

## Arrays

```
$ echo '[{"name":"A"},{"name":"B"}]' | jq '.[].name'
"A"
"B"

$ echo '[{"name":"A"},{"name":"B"}]' | jq '.[0].name'
"A"

$ echo '[1,2,3,4,5]' | jq '.[1:3]'         # slice
[2, 3]

$ echo '[1,2,3]' | jq 'length'
3
```

`.[]` iterates each element. `.[N]` indexes; `.[A:B]` slices.

## Selecting multiple fields

```
$ echo '{"name":"L","age":55,"city":"Helsinki"}' | jq '{name, age}'
{
  "name": "L",
  "age": 55
}

$ jq '{name: .name, country: .city}'        # rename
```

Build a new object by wrapping in `{...}`.

## Pipes inside jq

```
$ jq '.users[] | .name'                     # iterate users, get name of each
$ jq '.users | length'                      # count of users
$ jq '.users[] | select(.active)'           # only active users
$ jq '.users[] | select(.age > 25)'         # age > 25
```

Inside jq, `|` works like a Unix pipe but for JSON values.

## select — filter

```
$ jq '.items[] | select(.status == "active")'
$ jq '.items[] | select(.tags | contains(["urgent"]))'
$ jq '.[] | select(.amount > 1000)'
```

Common pattern: select then extract.

```
$ curl -s api.example.com/issues | jq '.[] | select(.state=="open") | .id'
```

= IDs of open issues.

## Combining outputs

`@csv` and `@tsv` format arrays:

```
$ jq -r '.users[] | [.name, .age] | @csv' users.json
"Linus",55
"Linda",30
```

Useful for piping JSON into spreadsheets or further processing.

## Transformations

```
$ jq '.users | map(.name)'                  # array of names
$ jq '.users | sort_by(.age)'               # sort by field
$ jq '.users | group_by(.team)'             # group by team
$ jq '.users | min_by(.age)'                # min by field
$ jq '[.users[] | .age] | add'              # sum ages
$ jq '.users | length'                      # count
```

`map`, `sort_by`, `group_by` — like programming, but in jq syntax.

## Modifying values

```
$ jq '.name = "Linus B Torvalds"'
$ jq '.tags += ["kernel"]'
$ jq 'del(.password)'                       # delete a field
$ jq '.user.age += 1'                       # increment
```

Returns the modified value to stdout. Use shell redirect to write to file.

## Reading from file

```
$ jq '.name' data.json
$ jq -s 'add' file1.json file2.json         # slurp into array, then add
```

`-s` slurps all input into a single array — useful for combining files.

## Practical scenarios

**Extract IDs from API response:**

```
$ curl -s api/users | jq -r '.[].id'
```

**Count items by status:**

```
$ jq '.items | group_by(.status) | map({status: .[0].status, count: length})'
```

**Pretty-print JSON:**

```
$ cat compressed.json | jq .
$ curl -s api/endpoint | jq .
```

`jq .` with no filter just re-formats nicely. The single most common use.

**Compact / minify:**

```
$ jq -c . input.json > minified.json
```

**Extract from kubernetes / docker:**

```
$ kubectl get pods -o json | jq '.items[] | {name: .metadata.name, status: .status.phase}'
$ docker inspect container | jq '.[0].State.Health.Status'
$ aws ec2 describe-instances | jq '.Reservations[].Instances[] | {id: .InstanceId, type: .InstanceType}'
```

JSON output from AWS/kubectl/docker → jq → exactly what you want.

## Error handling

```
$ echo 'null' | jq '.field?'
null

$ echo '{}' | jq '.field?'
null

$ echo '{}' | jq '.field // "default"'
"default"
```

`?` makes missing fields null instead of error. `//` is the "alternative" operator.

## When jq isn't enough

For very complex transformations, switch to Python with `json` module — much clearer for multi-step logic. jq is great for one-liners and pipeline stages; not for 50-line programs.

## Combining jq with other tools

```
$ kubectl get pods -o json | jq -r '.items[].metadata.name' | xargs -I{} kubectl logs {}
$ curl -s api/users | jq -r '.[] | [.name, .email] | @csv' | column -t -s,
$ docker ps --format '{{json .}}' | jq -s '.'        # turn line-by-line JSON into array
```

## Mistakes to avoid

- **Parsing JSON with grep / sed.** Use jq; otherwise you'll fail on whitespace variations, escaped quotes, nested objects.
- **Forgetting `-r`.** Output stays JSON-quoted; later pipes break.
- **No error handling.** Missing fields error out; use `?` or `//` for safety.
- **Trying to write multi-line scripts.** Switch to Python for complexity.

## Summary

- `jq` is the JSON Swiss Army knife for shells.
- `.` is input; `.field`, `.[]`, `.[0]` navigate; `|` pipes inside jq.
- `select`, `map`, `sort_by`, `group_by` for filtering and aggregation.
- `-r` for raw (no quotes) output.
- `@csv`, `@tsv` for tabular output.
- `?` and `//` for missing-field safety.
