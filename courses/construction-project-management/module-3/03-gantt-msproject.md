# Gantt charts + Microsoft Project

## What Gantt charts are

Horizontal bar chart:
- X-axis: time.
- Y-axis: activities.
- Each activity = bar from start to finish.
- Dependencies shown as arrows.

Named after Henry Gantt (1910s).

For: visual schedule.

## Reading Gantt

- **Bar length.** Duration.
- **Position.** Start + finish dates.
- **Color.** Often by phase or trade.
- **Critical path.** Highlighted (often red).
- **Progress.** Filled in as % complete.
- **Today line.** Current date marker.

For: schedule communication.

## Microsoft Project

Most common SMB tool:
- Activity list.
- Dependencies via predecessor column.
- Auto-calculate critical path.
- Resource assignment.
- Gantt view + network view.

Excel-like interface but task-aware.

For: standard PM software.

## Primavera P6

Enterprise standard:
- Multi-project management.
- Resource pools across projects.
- Advanced reporting.
- Complex constraints.
- Used: large GCs, government, infra.

Steeper learning curve.

For: large + complex projects.

## Procore + cloud tools

- Procore.
- Buildertrend (residential focus).
- CoConstruct.
- e-Builder (gov focus).

Mobile + collaboration + integrations.

For: field-driven modern.

## Schedule baseline

Save initial schedule as baseline:
- Compare actuals vs. plan.
- Variance reporting.

Re-baseline only with formal change.

For: tracking discipline.

## Schedule updates

Weekly or biweekly:
- % complete per activity.
- Adjust forecast.
- Identify slipped activities.
- Communicate to team.

Owner often requires monthly updates.

For: rhythm.

## Float ownership

Who owns float?
- Some contracts: owner.
- Some: contractor.
- Some: silent.

Affects who can use schedule slack.

For: contract specifics.

## 4D / 5D scheduling

- **4D.** Schedule + 3D BIM model. Visualize construction sequence over time.
- **5D.** Add cost dimension.

Synchro, Bentley SYNCHRO, BIM 360 used.

For: advanced visualization.

## Schedule analysis

Common metrics:
- Schedule variance (planned vs. actual).
- Schedule Performance Index (SPI).
- Time-related claims.

EVM applies here too.

For: performance tracking.

## Look-ahead schedule

3-week or 6-week look-ahead:
- Detailed near-term activities.
- Daily/weekly granularity.
- Coordinate trades.

Subset of master schedule.

For: tactical execution.

## Recovery schedule

When behind:
- Recovery plan with new sequence.
- May involve crash/fast-track.
- Owner approval if extends.

For: getting back on track.

## Mistakes to avoid

- **No baseline.** Can't measure progress.
- **Updating without analyzing.** Missing trends.
- **Skipping look-aheads.** Trades not coordinated.

## Summary

- Gantt = bar chart over time.
- MS Project (SMB), P6 (enterprise), Procore (cloud).
- Baseline → actuals → variance.
- 4D/5D adds model + cost dimensions.
- Look-ahead schedules tactical.
