# ERC-1155 multi-token

## What ERC-1155 solves

Single contract can hold:
- Fungible tokens (e.g., game gold, 100k copies).
- Semi-fungible tokens (e.g., 1000 copies of a sword).
- NFTs (unique tokenIds with supply 1).

All under one contract → cheaper deploy + batch operations.

For: gaming items, multi-edition collections.

## Interface

```solidity
interface IERC1155 {
    function balanceOf(address account, uint256 id) external view returns (uint256);
    function balanceOfBatch(address[] calldata accounts, uint256[] calldata ids) external view returns (uint256[] memory);
    function setApprovalForAll(address operator, bool approved) external;
    function isApprovedForAll(address account, address operator) external view returns (bool);
    function safeTransferFrom(address from, address to, uint256 id, uint256 amount, bytes calldata data) external;
    function safeBatchTransferFrom(address from, address to, uint256[] calldata ids, uint256[] calldata amounts, bytes calldata data) external;

    event TransferSingle(address indexed operator, address indexed from, address indexed to, uint256 id, uint256 value);
    event TransferBatch(address indexed operator, address indexed from, address indexed to, uint256[] ids, uint256[] values);
}
```

For: balance per (user, tokenId).

## OpenZeppelin ERC1155

```solidity
import "@openzeppelin/contracts/token/ERC1155/ERC1155.sol";

contract GameItems is ERC1155, Ownable {
    uint256 public constant GOLD = 0;
    uint256 public constant SWORD = 1;
    uint256 public constant SHIELD = 2;

    constructor() ERC1155("https://game.io/items/{id}.json") Ownable(msg.sender) {
        _mint(msg.sender, GOLD, 10**18, "");      // 1e18 gold
        _mint(msg.sender, SWORD, 1000, "");       // 1000 swords
        _mint(msg.sender, SHIELD, 100, "");       // 100 shields
    }
}
```

Multiple token types in one deploy.

For: cost-efficient multi-asset contracts.

## URI pattern

```
"https://game.io/items/{id}.json"
```

`{id}` replaced with hex-encoded tokenId. Clients query metadata per tokenId.

For: structured metadata serving.

## Batch operations

```solidity
// Mint multiple types in one call
function airdrop(address[] calldata recipients, uint256 id, uint256 amount) external onlyOwner {
    for (uint i; i < recipients.length;) {
        _mint(recipients[i], id, amount, "");
        unchecked { ++i; }
    }
}

// Transfer many tokens at once
function bulkTransfer(address to, uint256[] calldata ids, uint256[] calldata amounts) external {
    safeBatchTransferFrom(msg.sender, to, ids, amounts, "");
}
```

For: airdrops + multi-item shops.

## Receiver hooks

```solidity
function onERC1155Received(
    address operator,
    address from,
    uint256 id,
    uint256 value,
    bytes calldata data
) external returns (bytes4);
```

Contracts receiving ERC-1155 must implement this. Prevents accidental burns.

For: safe transfers to contracts.

## Approval model

Single approval covers ALL tokenIds. No per-token approval (unlike ERC-721's `approve`).

```solidity
gameItems.setApprovalForAll(marketplace, true);  // Approve marketplace for all items
```

Simplifies marketplace integration.

For: streamlined approval UX.

## Custom URI per token

```solidity
mapping(uint256 => string) private _tokenURIs;

function uri(uint256 tokenId) public view override returns (string memory) {
    if (bytes(_tokenURIs[tokenId]).length > 0) return _tokenURIs[tokenId];
    return super.uri(tokenId);
}

function setTokenURI(uint256 tokenId, string memory tokenURI) external onlyOwner {
    _tokenURIs[tokenId] = tokenURI;
}
```

For: heterogeneous metadata per token.

## ERC-1155 vs. ERC-721

| | ERC-721 | ERC-1155 |
|--|---------|----------|
| Unique per ID | Yes | Yes |
| Multiple copies per ID | No (each is unique) | Yes |
| One contract / many types | One contract per collection | Multiple types per contract |
| Batch ops | Manual | Built-in |
| Gas | Higher per token | Lower for batches |

For: choosing standard.

## Gas comparison

Batch 100 transfers:
- ERC-721: ~80k gas × 100 = 8M gas.
- ERC-1155 batch: ~500k gas for all 100.

16× cheaper for batches.

For: high-throughput use cases.

## Migration from 721 to 1155

Often easier to deploy new 1155 contract:
- Snapshot 721 holders.
- Distribute equivalent 1155 tokens.
- Burn or freeze 721.

Full bridge possible but complex.

For: legacy migration strategy.

## Mistakes to avoid

- **Single tokenId burn ≠ destroy contract.** Other tokenIds still active.
- **Forgetting onERC1155Received in receiving contracts.** Token stuck.
- **Hard-coded `{id}` in URI without server support.** Returns 404.
- **Treating fungible like NFT.** Approve-per-token confusion.

## Summary

- ERC-1155 = one contract, many token types (fungible + semi-fungible + NFT).
- Cheaper for batches; cheaper deploy.
- `setApprovalForAll` covers all tokenIds.
- Common in games + multi-tier collections.
