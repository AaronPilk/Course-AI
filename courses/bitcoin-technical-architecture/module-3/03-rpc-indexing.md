# RPC and indexing

## RPC overview

Bitcoin Core exposes JSON-RPC over HTTP:
- Default port: 8332 (mainnet), 18332 (testnet).
- Authentication: rpcuser/rpcpassword or cookie file.
- Commands: ~200 RPCs available.

For: programmatic access.

## Authentication

Cookie auth (default):
```
~/.bitcoin/.cookie  # Auto-generated per startup
```

Or persistent:
```ini
rpcauth=username:hash$salt   # generated by rpcauth.py script
```

For: secure access.

## Python example

```python
from bitcoinrpc.authproxy import AuthServiceProxy

rpc = AuthServiceProxy("http://user:pass@127.0.0.1:8332")
info = rpc.getblockchaininfo()
print(f"Height: {info['blocks']}")

# Watch a tx
tx = rpc.gettransaction("abc...")
print(f"Confirmations: {tx['confirmations']}")
```

For: scripting + integration.

## txindex

```ini
txindex=1
```

Stores tx index across full chain. Without it, can only look up wallet transactions.

Adds ~10 GB; slows IBD slightly. Required for explorers + indexers.

For: full tx lookups.

## Indexers

Bitcoin Core doesn't index everything. Indexers add:

- **Electrum Server (electrs).** Address balance lookups; for Electrum wallet.
- **Esplora.** Block explorer (mempool.space uses this).
- **Mempool.space backend.** Open source; rich API.
- **Fulcrum.** Faster than electrs; lower-spec.
- **BTCPay Server.** Payment processing.

For: app-specific data.

## Electrum protocol

```python
# Connect to electrs (port 50001 default)
import socket, json

def call(method, params):
    s = socket.socket()
    s.connect(("localhost", 50001))
    s.send(json.dumps({"id": 1, "method": method, "params": params}).encode() + b"\n")
    response = s.recv(10000)
    return json.loads(response)

balance = call("blockchain.address.get_balance", ["bc1q..."])
```

For: light client UX.

## REST API

Bitcoin Core has a limited REST API:
```bash
curl http://localhost:8332/rest/headers/5/<blockhash>.json
curl http://localhost:8332/rest/tx/<txid>.json
```

Read-only; no auth (firewall it).

For: simple integrations.

## Notifications

```ini
walletnotify=/path/to/script.sh %s
blocknotify=/path/to/script.sh %s
```

Calls script on new tx / block. Pass txid / blockhash as argument.

For: event-driven apps.

## ZeroMQ pub/sub

```ini
zmqpubrawtx=tcp://127.0.0.1:28332
zmqpubrawblock=tcp://127.0.0.1:28333
zmqpubhashtx=tcp://127.0.0.1:28334
zmqpubhashblock=tcp://127.0.0.1:28335
```

Subscribe via ZMQ socket. Used by Lightning, mempool.space, etc.

For: real-time event streams.

## Block explorer requirements

Full explorer like mempool.space:
- txindex=1.
- Electrs / esplora indexer.
- Backend (Node.js typically).
- Frontend (React typically).
- ~1 TB SSD recommended for performance.

For: building your own explorer.

## Mistakes to avoid

- **No txindex when needed.** Some calls fail.
- **RPC port open.** Wallet drained.
- **Trusting indexer data.** Verify against your own node when stakes high.

## Summary

- bitcoind exposes ~200 RPCs.
- txindex=1 for tx lookups outside wallet.
- electrs / esplora / mempool.space for indexing.
- ZMQ for event streams.
